## Layout Contract

### Push Permission Card (D-01, D-03)

- Position: top of TaskBoard content area, above the task list, below the Header
- Style: matches existing card aesthetic -- `bg-secondary` background, `rounded-lg`, `border border-border`, padding `16px`
- Layout: single row on mobile. Left: text content. Right: action buttons.
  - Text: "Get notified when new tasks come in" (Body 16px/400, `text-foreground`)
  - Buttons: "Enable" (accent `bg-blue-600 text-white`, `min-h-[44px]`), dismiss X icon button (`min-h-[44px] min-w-[44px]`)
- Max width: `960px` centered (matches task board container)
- Horizontal margin: `16px` on mobile, auto-centered on desktop
- Visibility: shown after first login if push permission is `'default'` (not yet asked). Hidden if `'granted'` or `'denied'`. Hidden if dismissed (stored in `localStorage`).
- On iOS not in standalone mode: hide push card, show install banner instead (iOS requires standalone for push -- Pitfall 1)

### Install Banner (D-07, D-08)

- Position: below push permission card (or at top of board if push card is hidden), above task list
- Style: same card aesthetic as permission card -- `bg-secondary`, `rounded-lg`, `border border-border`, padding `16px`
- Layout: single row. Left: text + optional iOS instructions. Right: action buttons.
  - Text: "Install Maison for quick access" (Body 16px/400, `text-foreground`)
  - Android: "Install" button triggers `beforeinstallprompt` (accent `bg-blue-600 text-white`, `min-h-[44px]`), dismiss X icon button
  - iOS: expandable step-by-step instructions (Label 14px/400, `text-muted-foreground`): "1. Tap the Share button 2. Scroll down and tap Add to Home Screen"
- Max width: `960px` centered
- Visibility: hidden in standalone mode (`display-mode: standalone`). Hidden if dismissed (stored in `localStorage`, keyed per device). Re-accessible from UserMenu.
- Dismissible: once per device. Dismiss X button stores flag in `localStorage`.

### Offline Banner -- Mid-Session (D-06)

- Position: fixed top banner, below the sticky Header, full width
- Style: `bg-amber-50`, `border-b border-amber-200`, padding `8px 16px`
- Layout: centered text, single line
  - Text: "Offline -- changes won't sync until reconnected" (Label 14px/400, `text-amber-800`)
- Animation: slide down on appear, slide up on disappear (CSS transition, `transform translateY`, 200ms ease)
- Auto-dismiss: disappears when `navigator.onLine` returns true (via `online` event)
- z-index: `z-40` (below Header's `z-50` if applicable, but visually below the sticky header)

### Offline Shell -- Full Load (D-05)

- Position: replaces the entire board content area (below Header)
- Style: centered vertically and horizontally within the content area
- Layout:
  - App icon: `pwa-192x192.png`, 64px rendered size, centered, `mb-4`
  - Heading: "You're offline" (Heading 20px/600, `text-foreground`)
  - Body: "Connect to the internet to see tasks." (Body 16px/400, `text-muted-foreground`, `mt-2`)
- Background: `bg-background` (consistent with normal app shell)
- The Header still renders (branded shell) -- only the content area shows the offline message

### UserMenu Extensions (D-02, D-04, D-08)

Extend existing `UserMenu.tsx` dropdown with new items:

```
[User Name]
[user@email.com]
─────────────────
Notifications off     (only if push is denied/dismissed)
Install app           (only if not standalone and not dismissed)
─────────────────
Sign Out
```

- "Notifications off" item: `text-muted-foreground` (Label 14px/400). On click:
  - If browser permission is `'default'`: trigger `Notification.requestPermission()`
  - If browser permission is `'denied'`: show explanatory text "Enable in browser settings" (no action, informational)
- "Install app" item: standard dropdown item text. On click: trigger install flow (Android) or show iOS instructions

### Push Notification (OS-level, D-09, D-10, D-11, D-12)

- Title: "New Task"
- Body: "{client_name} -- {request_type}"
- Icon: `/pwa-192x192.png`
- Badge: `/pwa-192x192.png`
- Click action: opens task board (`/`), focuses existing window if open
- Sound/vibration: OS defaults (no custom configuration)

## Interaction Patterns

### Push Permission Flow (NOTF-03, D-01)

1. Staff logs in and lands on task board
2. If `Notification.permission === 'default'` and not previously dismissed: PushPermissionCard appears at top of board
3. Staff taps "Enable": browser permission prompt appears (user gesture satisfies iOS requirement)
4. If granted: subscription created, sent to server, card disappears permanently
5. If denied: card disappears, "Notifications off" appears in UserMenu (D-02)
6. If dismissed (X button): card disappears, "Notifications off" appears in UserMenu, stored in `localStorage` (D-04)
7. Re-enabling from UserMenu: if permission is `'default'`, triggers permission prompt again. If `'denied'`, shows "Enable in browser settings" text.

### Install Flow (PWA-01, D-07, D-08)

1. App detects it is not running in standalone mode
2. InstallBanner appears below PushPermissionCard (or at top if push card hidden)
3. **Android:** Staff taps "Install" -> native `beforeinstallprompt` dialog appears -> if accepted, banner hides permanently
4. **iOS:** Banner shows step-by-step instructions inline -> staff follows manual steps -> on next launch in standalone mode, banner is hidden
5. If dismissed (X button): banner hides, flag stored in `localStorage`. "Install app" option remains in UserMenu.
6. If already standalone: banner never shows, "Install app" not shown in UserMenu

### Offline Detection (PWA-02, D-05, D-06)

1. **App loads offline:** Header renders (branded shell), OfflineShell replaces board content. No task data shown. No stale data risk.
2. **Connection drops mid-session:** OfflineBanner slides down below Header. Tasks remain visible (already loaded) but mutations will fail.
3. **Connection restored:** OfflineBanner slides up and disappears automatically. No user action needed.
4. **Offline + reconnect on load:** OfflineShell disappears, board loads normally when data fetch succeeds.

### Card Priority Order

When both cards are visible, they stack in this order at the top of the task board:

1. PushPermissionCard (higher priority -- D-01 specifies this is the primary prompt)
2. InstallBanner (below, only if applicable)
3. Task list begins

Each card disappears independently. When both are dismissed, task list occupies the full content area.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Focus visible | `ring-2 ring-ring ring-offset-2` on Enable, Install, and dismiss buttons |
| Touch targets | 44x44px minimum on all buttons including dismiss X icons |
| Dismiss buttons | `aria-label="Dismiss notification prompt"` and `aria-label="Dismiss install prompt"` |
| Offline banner | `role="status"` with `aria-live="polite"` so screen readers announce connectivity changes |
| Offline shell | Heading is `h2` (page still has Header `h1`), descriptive body text |
| Push permission card | `role="region"` with `aria-label="Push notification setup"` |
| Install banner | `role="region"` with `aria-label="App installation"` |
| iOS instructions | Ordered list (`ol`) for step-by-step instructions |
| Color contrast | `text-amber-800` on `bg-amber-50` = 7.5:1 (passes AA). `text-muted-foreground` on `bg-secondary` passes AA. White on `bg-blue-600` = 4.7:1 (passes AA). |
