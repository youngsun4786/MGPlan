  <what-built>
Complete follow-up reminders system: SQL migration (last_reminder_at column, get_stale_open_tasks RPC, pg_cron job), remind-tasks Edge Function, and unit tests.
  </what-built>
  <how-to-verify>
**Pre-requisites (one-time setup):**

1. Apply migration to Supabase:
   - Run `supabase db push` OR copy contents of `supabase/migrations/0004_reminder_cron.sql` into Supabase Dashboard > SQL Editor and execute

2. Deploy the Edge Function:
   - Run `supabase functions deploy remind-tasks`

3. Set Vault secrets in Supabase SQL Editor:
   ```sql
   SELECT vault.create_secret('https://<your-project-ref>.supabase.co', 'project_url');
   SELECT vault.create_secret('<your-service-role-key>', 'service_role_key');
   ```

4. Set REMINDER_THRESHOLD_HOURS env var:
   - Run `supabase secrets set REMINDER_THRESHOLD_HOURS=2`
   - Or set via Dashboard > Edge Functions > remind-tasks > Environment Variables

**Verification steps:**

1. **Check pg_cron job exists:** In SQL Editor, run:
   ```sql
   SELECT * FROM cron.job WHERE jobname = 'remind-stale-tasks';
   ```
   Expected: One row with schedule `*/15 * * * *`

2. **Test Edge Function directly:** In SQL Editor or via curl:
   ```sql
   SELECT net.http_post(
     url := '<your-project-url>/functions/v1/remind-tasks',
     headers := jsonb_build_object(
       'Content-Type', 'application/json',
       'Authorization', 'Bearer <your-service-role-key>'
     ),
     body := '{}'::jsonb
   );
   ```
   Check Edge Function logs in Dashboard for output.

3. **Create a test task** in the app with Open status, then wait for the reminder (or temporarily set REMINDER_THRESHOLD_HOURS=0 to trigger immediately).

4. **Check cron run history** after 15 minutes:
   ```sql
   SELECT * FROM cron.job_run_details ORDER BY start_time DESC LIMIT 5;
   ```
   Expected: status = 'succeeded'

5. **Confirm push notification received** on a subscribed device with title "Overdue Task" and body "{client_name} has been waiting {elapsed_time}"
  </how-to-verify>

    - Push notification received with title "Overdue Task" and body containing client name and elapsed time
  </acceptance_criteria>
