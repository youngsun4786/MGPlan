2. **formatElapsed function** -- export for testability. Calculates elapsed time from `created_at` (total wait time, NOT from last_reminder_at). Per D-05, human-friendly format:
   - `>= 1 day`: "1 day" or "N days"
   - `>= 1 hour`: "1 hour" or "N hours"
   - `< 1 hour`: "1 minute" or "N minutes"

```typescript
export function formatElapsed(createdAt: string): string {
  const diffMs = Date.now() - new Date(createdAt).getTime()
  const minutes = Math.floor(diffMs / 60_000)
  const hours = Math.floor(minutes / 60)
  const days = Math.floor(hours / 24)

  if (days > 0) return days === 1 ? '1 day' : `${days} days`
  if (hours > 0) return hours === 1 ? '1 hour' : `${hours} hours`
  return minutes === 1 ? '1 minute' : `${minutes} minutes`
}
```

3. **Deno.serve handler** (no auth check -- invoked by pg_cron with service role key, per Pitfall 4 in RESEARCH):

   a. Read threshold: `const thresholdHours = Number(Deno.env.get('REMINDER_THRESHOLD_HOURS') ?? '2')` (per D-09, default 2)

   b. Query stale tasks via RPC: `supabase.rpc('get_stale_open_tasks', { threshold_hours: thresholdHours })`

   c. If no stale tasks or error, return `{ reminded: 0 }` with 200 status.

   d. Fetch all push subscriptions: `supabase.from('push_subscriptions').select('endpoint, p256dh, auth')`

   e. If no subscriptions, return `{ reminded: 0, reason: 'no_subscribers' }` with 200 status.

   f. For each stale task:
      - Build notification payload per D-04: `{ title: 'Overdue Task', body: '${task.client_name} has been waiting ${formatElapsed(task.created_at)}' }`
      - Fan out to all subscribers using `Promise.allSettled` (same pattern as notify-task)
      - Clean up expired subscriptions (410 Gone) -- same pattern as notify-task
      - Update `last_reminder_at` per D-07: `supabase.from('tasks').update({ last_reminder_at: new Date().toISOString() }).eq('id', task.id)`

   g. Return `{ reminded: totalSent }` with 200 status.

   h. Wrap entire handler in try/catch, log errors with `console.error('remind-tasks error:', err)`, return 500 on failure.

Keep the same logging style as notify-task for consistency.
