## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap between badge and text |
| md | 16px | Default element spacing, form field gaps (`space-y-4`) |
| lg | 24px | Section padding, dialog body padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets use `min-h-[44px] min-w-[44px]` per existing convention (44px, not a multiple of 4 but required by WCAG 2.5.8).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 14px (text-sm) | 400 (normal) | 1.5 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Caption | 12px (text-xs) | 400 (normal) | 1.5 |

2 weights declared: 400 (normal) for body, labels, and captions; 600 (semibold) for headings. Labels are distinguished from body text by context and positioning (above form fields), not by weight.

Source: Matches existing codebase. Header uses `text-xl font-semibold`, body uses `text-base`/`text-sm font-normal`, metadata uses `text-xs`.

## Interaction States

### Upload Flow (Happy Path)

1. Staff taps "+" in header -> `DropdownMenu` appears with two items
2. Staff selects "Upload Screenshot" -> file picker opens (`accept="image/*"`)
3. On mobile: camera capture or gallery selection available
4. File selected -> client-side validation (type + size) -> resize to max 2048px longest edge
5. Task form dialog opens immediately with skeleton shimmer fields (D-03)
6. `ScreenshotPreview` appears above form (collapsed on mobile, expanded on desktop)
7. Server processes image via Claude vision API
8. Fields fill in progressively as extraction completes
9. Low-confidence fields show amber border + warning icon (D-04)
10. Staff reviews, corrects if needed, taps "Create Task"

### Upload Flow (Error States)

| Scenario | Behavior | Toast Copy |
|----------|----------|-----------|
| Unreadable screenshot (D-06) | Open empty form | "Couldn't read the screenshot. Enter details manually." |
| Partial extraction (D-07) | Fill extracted fields, leave rest empty, amber on low-confidence | No toast |
| API timeout (D-08) | Open empty form | "Screenshot processing timed out. Enter details manually." |
| Rate limit hit (D-13) | Open empty form | "Screenshot limit reached (10/hour). Try again soon." |
| File too large (>10MB) | Block upload, no form | "Image too large. Maximum size is 10MB." |
| Invalid file type | Block upload, no form | "Unsupported format. Use PNG, JPEG, HEIC, or WebP." |

### Skeleton Form State

- Each form field (client_name, phone, service, request_type, preferred_datetime, notes) renders a `Skeleton` rectangle matching the field's dimensions
- Skeleton uses shadcn `Skeleton` component (neutral shimmer animation)
- Fields transition from skeleton to populated value with no layout shift
- Form is non-interactive during skeleton state (submit button disabled, shows "Processing...")

### Screenshot Preview

- `Collapsible` component above form fields
- Trigger button: "View Screenshot" / "Hide Screenshot" with chevron icon
- Desktop: expanded by default
- Mobile (< 640px): collapsed by default
- Image rendered at max-width 100%, aspect ratio preserved, rounded corners (`rounded-lg`)

### Screenshot Thumbnail on TaskRow

- 20x20px `Image` icon (lucide) rendered inline after client name, only when `screenshot_url` is present
- On tap: opens `ImageLightbox` (full Dialog with image at max viewport dimensions)
- Icon color: `text-slate-400` to avoid visual noise

### Dropdown Menu on "+" Button

- Button retains `bg-blue-600 hover:bg-blue-700 text-white min-h-[44px]` styling
- Dropdown items:
  - "Upload Screenshot" with `Camera` icon (lucide)
  - "Create Manually" with `PenLine` icon (lucide)
- Each dropdown item has `min-h-[44px]` touch target

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| < 640px (mobile) | "+" button shows icon only (existing). Dropdown menu full-width items. Screenshot preview collapsed by default. File input includes `capture="environment"` for camera access. Dialog full-screen. |
| >= 640px (desktop) | "+" button shows icon + label. Dropdown menu standard width. Screenshot preview expanded by default. Drag-and-drop zone visible in upload dialog. Dialog max-width 448px (`sm:max-w-md`). |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | Dropdown menu fully keyboard navigable (Radix built-in). Escape closes dropdown and dialogs. |
| Screen reader | Low-confidence fields announce via `aria-describedby` linking to warning text "AI extracted this -- please verify". Screenshot thumbnail has `aria-label="View attached screenshot"`. |
| Touch targets | All interactive elements maintain `min-h-[44px] min-w-[44px]`. |
| Focus management | When form opens with skeleton state, focus moves to first field once extraction completes (or immediately on error fallback). |
| Color contrast | Amber warning border (`amber-400`) on white background meets 3:1 minimum for non-text contrast. Amber text (`amber-800`) on amber background (`amber-50`) meets 4.5:1 for body text. |
