# Phase 1 — UI Design Contract

> Visual and interaction contract for Foundation & Core Board. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (to be initialized during scaffolding) |
| Preset | default (Tailwind utility-first, New York style) |
| Component library | Radix UI (via shadcn primitives) |
| Icon library | Lucide React |
| Font | System font stack (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, "Noto Sans", sans-serif`) |

### shadcn Components Required

| Component | Usage |
|-----------|-------|
| Button | Primary CTA, status transitions, form submit, dialog actions |
| Input | All text fields in task form and login form |
| Select | Request type dropdown, service field |
| Dialog | Delete task confirmation |

shadcn must be initialized during project scaffolding (Wave 0). Run `npx shadcn@latest init` after TanStack Start project setup. Then add components:

```bash
npx shadcn@latest add button input select dialog dropdown-menu label badge
```

## Spacing Scale

Declared values (8-point scale, all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between badge text and edge |
| sm | 8px | Compact element spacing, gap between form error and input |
| md | 16px | Default element spacing, form field vertical gap, card padding |
| lg | 24px | Section padding, task list item vertical padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page horizontal padding on desktop |
| 3xl | 64px | Login card vertical centering offset |

Exceptions: 44px minimum touch target on all interactive elements (buttons, status badges, dropdown triggers, task row tap areas). Use `min-h-[44px] min-w-[44px]` per CONVENTIONS.md.

## Color

| Role | Value | Tailwind Token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#ffffff` | `white` | Page background, form backgrounds, task board background |
| Secondary (30%) | `#f8fafc` | `slate-50` | Login page background, task row hover state, card surfaces, input backgrounds |
| Accent (10%) | `#2563eb` | `blue-600` | Primary CTA buttons, active links, focus rings |
| Destructive | `#dc2626` | `red-600` | Delete button, destructive confirmation, form validation errors |

### Accent Reserved For

- "Create Task" button (primary CTA)
- Login "Sign In" button
- Focus ring on inputs and interactive elements (`ring-blue-600`)
- "In Progress" status badge background

Accent is NOT used for: general text, borders, backgrounds, icons, or decorative elements.

### Status Badge Colors

| Status | Background | Text | Tailwind Classes |
|--------|------------|------|------------------|
| Open | `#f59e0b` / `amber-500` | `white` | `bg-amber-500 text-white` |
| In Progress | `#2563eb` / `blue-600` | `white` | `bg-blue-600 text-white` |
| Done | `#6b7280` / `gray-500` | `white` | `bg-gray-500 text-white` |

### Interactive States

| State | Treatment |
|-------|-----------|
| Hover (button) | Darken by one shade (e.g., `hover:bg-blue-700`) |
| Hover (task row) | `bg-slate-50` background |
| Focus | `ring-2 ring-blue-600 ring-offset-2` |
| Disabled | `opacity-50 cursor-not-allowed` |
| Done task row | `opacity-60` on the entire row (D-04: visually dimmed, not hidden) |

## Layout Contract

### Login Page (D-09)

- Centered card on `slate-50` background
- Card: `white` background, `rounded-lg`, `shadow-sm`, max-width `400px`, padding `32px`
- App name in Display size (28px/600) at top of card
- Email input, Password input, Sign In button (full-width, accent blue)
- No full-screen backgrounds, no illustrations

### Task Board (D-01, D-02, D-03)

- Single scrollable list feed, not kanban columns
- Sorted oldest first (FIFO queue) — D-02
- Each row displays: client name (Body 16px/400), phone number (Label 14px/500), request type (Label 14px/500), status badge
- Row minimum height: 44px (touch target)
- Row padding: 16px horizontal, 12px vertical
- Row separator: 1px `border-b border-slate-200`
- Done rows: `opacity-60` (D-04)

### Task Form

- Modal dialog for both create and edit (mobile-friendly, shadcn Dialog)
- Field order: Client Name, Phone Number, Service Requested, Request Type (Select), Preferred Date/Time, Notes
- Required fields marked with red asterisk: Client Name, Phone Number, Request Type
- Field-level error messages in `red-600` at `14px`, 8px below the input
- Submit button: full-width at bottom of form, accent blue
- Cancel button: ghost variant, left of submit

### Header / Navigation

- Sticky top bar, `white` background, `border-b border-slate-200`
- Left: App name (Heading 20px/600)
- Right: "Create Task" button (accent, with Plus icon from Lucide) + User avatar/initials dropdown (D-13)
- Height: 56px (accommodates 44px touch targets with 6px vertical padding)

### Responsive Breakpoints

| Breakpoint | Width | Layout Adjustment |
|------------|-------|-------------------|
| Mobile | 375px - 767px | Full-width task rows, stacked form fields, bottom-sticky CTA |
| Tablet | 768px - 1023px | Same as mobile with wider margins |
| Desktop | 1024px+ | Max-width container (960px), centered |

## Interaction Patterns

### Status Transitions (TASK-04)

- Tap status badge on task row to cycle: Open -> In Progress -> Done
- Badge acts as a button (44px min touch target)
- Optimistic update: badge changes immediately, reverts on server error
- No confirmation dialog for status changes (lightweight, reversible)

### Task Creation (TASK-01)

- Tap "Create Task" button in header
- Dialog opens with empty form
- Fill fields, tap "Create Task" submit button
- Dialog closes on success, new task appears in list via Realtime
- On validation error: field-level errors appear, dialog stays open

### Task Edit (TASK-05)

- Tap task row to open edit dialog (same form as create, pre-filled)
- Modify fields, tap "Save Changes" submit button
- Dialog closes on success, task updates in list via Realtime

### Task Delete (TASK-06)

- Inside edit dialog or via row action: tap delete icon/button
- Confirmation dialog appears (destructive pattern)
- "Delete" button in `red-600`, "Cancel" in ghost variant
- On confirm: dialog closes, task removed from list via Realtime

### Sign Out (D-13)

- Tap avatar/initials icon in top-right header
- DropdownMenu opens with "Sign Out" option
- Tap "Sign Out": session cleared, redirect to `/login`

### Loading States

- Initial board load: skeleton rows (3-4 placeholder rows with pulsing animation)
- Form submit: button shows loading spinner, disabled state
- Status change: optimistic badge update (no loading indicator needed)

### Empty State

- Centered in board area when no tasks exist
- Illustration: none (keep minimal)
- Heading: "No tasks yet" (Heading 20px/600, `text-slate-600`)
- Body: "Create your first task to start tracking client requests." (Body 16px/400, `text-slate-500`)
- Optional: subtle arrow or highlight pointing to "Create Task" button

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Focus visible | `ring-2 ring-blue-600 ring-offset-2` on all interactive elements |
| Touch targets | 44x44px minimum on all buttons, badges, links, row tap areas |
| Color contrast | All text meets WCAG AA: white on blue-600 = 4.7:1, white on amber-500 = 3.0:1 (use bold text for amber badge to compensate), white on gray-500 = 4.6:1 |
| Form errors | Field-level, associated via `aria-describedby`; error text in `red-600` |
| Dialog | shadcn Dialog includes focus trap, Escape to close, `aria-modal` |
| Screen reader | Status badges include `aria-label` with full status text |
| Keyboard nav | Tab order follows visual order; Enter/Space activates buttons |
